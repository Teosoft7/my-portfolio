# Data Science 포트폴리오 프로젝트 구조 요약 (summary.md)

이 문서는 미국 IT 업계 데이터 사이언스(Data Science) 및 머신러닝 엔지니어(MLE) 취업을 위한 포트폴리오 웹사이트의 기본 구조를 분석하고 요약한 것입니다.

## 🏗️ 1. 전체 아키텍처 및 기술 스택
현재 프로젝트는 **프론트엔드(Next.js)와 백엔드(FastAPI)가 분리된 모노레포(Monorepo) 구조**를 가지고 있으며, **Docker Compose**를 통해 컨테이너화되어 있습니다. 미국 테크 기업에서 선호하는 모던 웹 스택과 풀스택 엔지니어링 역량을 보여주기에 매우 적합한 뼈대입니다.

- **Frontend:** Next.js (React 기반, App Router 사용) - SSR(서버 사이드 렌더링) 및 SEO 최적화에 유리.
- **Backend:** FastAPI (Python) - 데이터 사이언스 모델 서빙 및 비동기 처리에 강력한 최신 웹 프레임워크.
- **Infrastructure:** Docker & Docker Compose - 환경 의존성 문제 해결 및 일관된 실행 환경 제공.

---

## 📂 2. 주요 디렉토리 및 파일 분석

### 🖥️ Frontend (`/frontend`)
사용자에게 보여지는 UI 및 데이터 시각화를 담당하는 Next.js 애플리케이션입니다.

- **`app/`**: Next.js 13 이상의 App Router 구조입니다.
  - `page.tsx` & `layout.tsx`: 메인 홈페이지와 전체 레이아웃을 정의합니다.
  - `projects/`: 개별 포트폴리오 프로젝트 상세 페이지들이 위치할 라우팅 폴더입니다.
- **`components/`**: 재사용 가능한 UI 컴포넌트(네비게이션 바, 차트, 데이터 테이블 등)를 모아두는 곳입니다.
- **`content/`**: 프로젝트 설명, 블로그 포스트 등을 위한 정적 콘텐츠(Markdown, MDX 등)를 저장하기 적합한 폴더입니다.
- **`package.json`**: 프론트엔드 의존성(Node.js 패키지)을 관리합니다.

### ⚙️ Backend (`/backend`)
데이터 전처리, 머신러닝 모델 추론 및 비즈니스 로직을 처리하는 API 서버입니다.

- **`app/`**: FastAPI 메인 애플리케이션 코드가 위치합니다.
  - `main.py`: FastAPI 서버의 진입점. CORS 설정 및 API 라우터를 등록합니다.
  - `api/routes.py`: 프론트엔드와 통신할 API 엔드포인트들을 정의합니다. (현재 `calculate-ooh-index` 예제 API가 존재)
  - `core/`: 애플리케이션 설정, 보안, 공통 유틸리티 등을 관리합니다.
  - `services/`: 핵심 비즈니스 로직 및 데이터 알고리즘 모듈입니다. (현재 `index_engine.py`에 OOH 지수 산출 로직 존재)
- **`requirements.txt`**: 백엔드 구동에 필요한 Python 패키지 의존성을 관리합니다 (pandas, scikit-learn 등이 추가될 예정).
- **`Dockerfile`**: 백엔드 서버를 위한 Docker 이미지 빌드 설정입니다.

### 🛠️ Root 레벨
- **`docker-compose.yml`**: 프론트엔드(포트 3000)와 백엔드(포트 8000) 서비스를 묶어서 한 번에 실행하게 해주는 설정 파일입니다.
- **`.github/`**: GitHub Actions 등을 활용한 CI/CD 파이프라인(자동 테스트 및 배포) 설정이 들어갈 폴더입니다.

## 🚀 3. 미국 IT 취업을 위한 향후 개발 방향 (Next Steps)

현재 뼈대는 훌륭하게 잡혀 있습니다. 앞으로 살을 붙여나갈 때 미국 DS 시장에서 어필할 수 있는 포인트들은 다음과 같습니다.

1. **데이터 파이프라인 및 모델 서빙 구현 (`backend/services`)**:
   - 단순한 수식 계산(`index_engine.py`)을 넘어, 실제 머신러닝 모델(Pickle, ONNX 등)을 로드하여 예측값을 반환하는 엔드포인트를 추가하세요.
2. **인터랙티브 데이터 시각화 (`frontend/components`)**:
   - Recharts, Plotly.js, 혹은 D3.js를 활용하여 백엔드에서 받아온 분석 결과를 사용자가 조작해 볼 수 있는 동적인 대시보드를 구축하세요. (예: 파라미터 조절에 따른 예측 결과 변화 시각화)
3. **비즈니스 임팩트 중심의 스토리텔링 (`frontend/content`)**:
   - 미국 채용 시장은 기술 그 자체보다 **'어떤 문제를 풀었고, 비즈니스 가치를 얼마나 창출했는지'**를 중요하게 봅니다. 각 프로젝트 페이지(`projects/`)에 문제 정의, 데이터 수집/전처리, 모델 선정 이유, 성과 지표(Metric)를 명확히 작성하세요.
4. **CI/CD 및 클라우드 배포 (`.github/`)**:
   - 로컬 Docker 환경을 넘어, AWS(EC2/ECS)나 GCP, 혹은 Vercel(Front) + Render/Heroku(Back) 등의 클라우드 서비스로 배포 자동화 파이프라인을 구축하면 엔지니어링 역량을 크게 돋보이게 할 수 있습니다.

**결론적으로,** 프론트엔드와 백엔드를 나누고 API로 통신하는 현재의 아키텍처는 **확장 가능하고 실무 친화적인 훌륭한 출발점**입니다. 하나씩 프로젝트와 모델을 추가하며 포트폴리오를 완성해 나가시길 바랍니다!
